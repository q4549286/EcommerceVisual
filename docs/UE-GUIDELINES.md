# UE 交互规范

## 1. 核心路径

1. 上传商品图
2. 填写商品名称、类目、描述
3. 选择文案语言、视觉风格、图片类型
4. 选择提示词模式或直接生图模式
5. 点击生成
6. 查看结果，复制提示词或下载图片

## 2. 上传

- 点击上传区或拖拽到上传区都会触发文件选择
- 仅接受 JPG、PNG、WEBP 三种格式
- 上传后立即展示预览，提供「重新选择」替换

## 3. 选项

- 文案语言、视觉风格、图片类型均为按钮形式
- 视觉风格为单选
- 图片类型为多选，至少选择一项
- 选中态采用黑底白字，未选采用白底灰边

不再提供目标平台选择，避免出现无意义的选项。

## 4. 表单校验

点击生成时校验以下字段：

- 商品图
- 商品名称
- 商品类目
- 商品描述
- 至少一种图片类型

未通过时在按钮上方显示红色提示。

## 5. 生成模式

- 提示词模式：仅生成方案与提示词，不调用接口
- 直接生图模式：调用服务端 `/api/generate`，需要 API Key

未配置 API Key 时直接生图会失败，但提示词方案仍会保留。

## 6. 结果

结果区每张图为一个卡片，依次展示：

- 标题、用途、尺寸
- 图片预览（提示词模式不出图）
- 本地化文案
- 设计说明
- 完整提示词与禁止项
- 复制提示词、复制文案、下载图片三个操作

## 7. 错误

错误以红色文案显示原因。直接生图失败时保留提示词方案，便于直接复制使用。

## 8. 移动端

- 单栏纵向滚动
- 可点击区域不小于 44px
- 表单分组与结果卡片纵向叠放
